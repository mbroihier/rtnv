Test Case INIT-0020
System Test PHS
Title Initialization with an existing SQLite database - no entries
Objective This test case is intended to verify the following implied or specified requirements:
  • When the app is started by selecting from the Android app selection menu, the app shall do the following:
  • If the portfolio database does exist, the app shall query the public server to retrieve and store the latest closing price for each stock ticker in the portfolio and then display a blanked main menu.
The objective of this test case is to verify the above requirements. This test will be executed following the first time the app is loaded onto the Android device. At that point, a portfolio will not exist so no database creation will be necessary.
Setup
  • Android device with version 4.2 or greater is attached to the Android development system. There is no copy of rtnv installed on the device.
Actions
  1. Compile and load rtnv onto the target Android device.
  2. Verify the compile completes with no errors.
  3. Verify the installation completes with no errors.
  4. Verify that a screen is displayed that has a text field for the ticker symbol, a text field for the number of shares, and a text field for the value of each share.
  5. Verify that there are four buttons: Enter, Delete, Next, and Calculate Net Value.
  6. Verify that no runtime errors are splashed to the screen.
  7. Press the home button.
  8. Press the task list button.
  9. Terminate the rtnv process.
  10. From the Android main menu, select rtnv to restart the application.
  11. Verify that a screen is displayed that has a text field for the ticker symbol, a text field for the number of shares, and a text field for the value of each share.
  12. Verify that there are four buttons: Enter, Delete, Next, and Calculate Net Value.
  13. Verify that no runtime errors are splashed to the screen.
  14. Press the Next button.
  15. Verify that nothing is displayed.
  16. Select the ticker text field.
  17. Enter the value "AAA"
  18. Select the number of shares field.
  19. Enter "1".
  20. Press the enter button.
  21. Verify that the fields are blanked out.
  22. Press the next button and verify that "AAA" is displayed in the ticker text field, "1" is displayed in the number of shares field, and "Not found" is displayed in the value of each share field.
Expected This being the second time the app has run on the device, the database should be there but empty. Pressing the next button the first time shows that. Pressing the next button after an entry is inserted then displays a stock entry, but since the ticker is not a valid stock ticker, no value for the stock can be displayed.
Results PASSED
Cleanup Unnecessary
Requirements RTNV-PR-INIT-010, RTNV-PR-INIT-010.002
